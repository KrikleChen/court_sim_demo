# Court Sandbox Child Sim Demo（像素文本版）

这是一个可以持续迭代的皇室演进玩法 demo，核心目标是把“无聊随机模拟”改造成“可决策、可叙事、有胜负线”的回合体验。

- 纯文字 + 像素字符地图
- 3 位宫廷角色（大皇子、二皇子、三公主）
- 每日自主行动与事件触发
- 奏报/密报系统（含可信度与偏差）
- 延迟记忆系统
- 皇帝逐人干预（交互模式可对每位角色单独下令）
- 每日事件卡（叙事事件）+ 资源预算系统
- 继承力评估面板（忠诚 / 威胁 / 功绩 / 继承力）

## 玩法目标（内置胜负目标）

默认规则会为每位孩子实时计算“继承力”：

- 忠诚：和皇帝的关系（信任、亲近、依赖）  
- 威胁：野心、求权、嫉妒、压力等因素综合  
- 功绩：智慧、武艺、政治、威望  

游戏结局会按以下逻辑给出方向性结论：

- 有人 `忠诚>=72 && 威胁<=52 && 功绩>=45` 时，标记为可继承候选
- 有人 `威胁>=82 && 忠诚<=44` 时，出现叛逆风险告警
- 最高“继承力”角色会作为最终讨论方向

这套机制让每一回合有明确的长期压力：不是只修数值，而是看能否把可控角色拉回“继承候选”。

## 资源系统（每回合预算）

皇帝每天可用的资源会自然恢复：

- `命令点（command_points）`
  - 每日 +2，封顶 12
  - 大部分决策的硬性消耗
- `谍报点（intel）`
  - 每日 +1，封顶 6
  - 叙事事件“调查”类选项消耗

示例：预算不足时可在交互界面看到 `[预算不足]`，将无法执行该动作。

## 干预体系

### 自动模式

- 维持原有按系统候选自动执行最高优先级一条

### 互动模式

- 每位角色逐一展示皇帝可选动作（可跳过）
- 逐人下令，可针对不同孩子施加不同处理
- 一回合可执行 0~N 条（只要资源与冷却允许）

## 叙事事件（新增）

每日中后段会生成 0~1 个“宫廷事件”（按压力、嫉妒、随机触发）：

- 宫中争执
- 夜闻异动
- 秘闻传书

每个事件附带 3 个处理选项（含消耗），处理后会直接影响人物关系、压力、忠诚与威胁状态，并记录在日志。

## 运行

```bash
cd /Users/ymatrix/court_child_sim_demo
python demo.py
```

默认是自动模式（不需每回合手动输入）。

手动模式:

```bash
python demo.py --interactive
```

可调参数:

- `--days` 天数（默认 100）
- `--seed` 随机种子（默认 20260616）
- `--interactive` 手动干预输入
- `--log` 日志路径（JSONL）

```bash
python demo.py --days 60 --seed 7 --interactive --log /tmp/court_sim.log
```

## 日志说明

日志文件每行一个 JSON，便于回放分析：

- `run_start`：开始配置（含初始资源）
- `day_start`：每日日始快照
- `day_child`：每个角色当日行为与状态变化
- `day_reports`：每个角色当日奏报
- `day_scoreboard`：每日政局面板（忠诚/威胁/功绩/继承力）与资源
- `day_incident`：今日叙事事件与处理结果
- `day_options`：自动策略候选
- `day_action`：皇帝当日干预（多条）
- `day_end`：每日日终角色状态
- `day_end_rollup`：汇总与资源
- `run_end`：终盘评分、最终结论、资源与角色总结
